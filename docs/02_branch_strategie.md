# Branch Strategy

We use GitHub Flow as our branching strategy.

## How it works

- `main` is always the stable branch
- for new features or fixes create a branch from main (e.g. `feat/something` or `fix/something`)
- when done open a pull request into main
- after review merge the pr and delete the branch

## Branch naming

- `feat/` for new features
- `fix/` for bug fixes
- `docs/` for documentation changes

## Rules

- never push directly to main
- always use pull requests
- keep branches short lived
